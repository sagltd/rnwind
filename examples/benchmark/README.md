# rnwind benchmark

Small automated benchmark comparing **rnwind**, **nativewind v5**, and **uniwind** on an identical scenario:

- `<View>` grid rendered at three sizes: **N=10, N=1,000, N=10,000**
- Same className per item: `w-32 h-24 rounded-2xl bg-primary items-center justify-center`
- Same React / React Native / Expo versions across all three apps

Modeled after [uni-stack/uniwind-benchmarks](https://github.com/uni-stack/uniwind-benchmarks), extended with a multi-size sweep so per-item scaling cost is visible on top of each library's fixed startup cost.

## TL;DR — run everything

```sh
# one-time setup
bun install

# build-time bench (fully automated, no simulator)
bun run --cwd examples/benchmark bench

# render bench — one app at a time on a simulator / device / browser
bun run --cwd examples/benchmark/apps/rnwind       ios --reset-cache
bun run --cwd examples/benchmark/apps/nativewind5  ios --reset-cache
bun run --cwd examples/benchmark/apps/uniwind      ios --reset-cache
```

## Layout

```
examples/benchmark/
├── bench.ts                   # build-time orchestrator
├── REPORT.md                  # generated by `bun run bench`
├── apps/
│   ├── rnwind/                # minimal Expo app using rnwind (workspace)
│   ├── nativewind5/           # same app using nativewind 5.0.0-preview.3
│   └── uniwind/               # same app using uniwind 1.6.x
└── packages/benchmark/
    └── src/
        ├── config.ts          # SIZES, RUNS_PER_SIZE, DELAY_BETWEEN_RUNS
        └── use-benchmark.ts   # shared useBenchmark hook
```

Each app is a minimal Expo project: `App.tsx`, `index.ts`, `metro.config.js`, `global.css`, `package.json`. Everything non-essential (routing, safe areas, assets) is stripped.

## Scripts

### `bun run --cwd examples/benchmark bench`

**What it does:** runs `expo export --platform web --clear` in each of the three apps, wall-clock-times the export, then walks the emitted `.bundle/` directory and sums up every `.js` / `.hbc` file size.

**What it produces:** [`REPORT.md`](./REPORT.md) with a table like:

| Library | Bundle time | JS size | vs rnwind |
|---|---:|---:|---:|
| rnwind | 8.33 s | 330.1 KB | 1.00× |
| nativewind5 | 9.13 s | 560.6 KB | 1.70× |
| uniwind | 10.33 s | 1063.6 KB | 3.22× |

**When to run:** any time you want a fast, reproducible signal on build cost and shipped JS weight. Fully automated — no device / simulator needed.

**Flags:**
- `bun run --cwd examples/benchmark bench:report-only` — skip the expo exports, just re-generate the report from whatever bundles are already in each `apps/*/.bundle/`.

### `bun run --cwd examples/benchmark/apps/<lib> ios` (or `android` / `web`)

**What it does:** launches the Expo dev server for one of the apps. When the app mounts, the shared `useBenchmark('<lib>')` hook takes over:

1. Starts at `SIZES[0]` (default 10 items)
2. Forces 5 timed re-renders of the grid, measuring wall-clock from `setRenderKey` to the first `requestIdleCallback` after layout
3. Prints one tagged line to Metro's stdout:
   ```
   RNWIND_BENCH_RESULT rnwind {"size":10,"runs":[7.8, 4.2, ...],"average":..., "min":..., "max":..., "median":..., "stdDev":...}
   ```
4. Advances to the next size and repeats until the sweep is done
5. Shows a summary table on-screen and prints three tagged lines total (one per size)

**What it measures:** real on-device render time per library, per element count. Scales N so per-item overhead shows up.

**When to run:** when you want the "on-device" numbers like the uniwind-benchmarks repo publishes. Do one app at a time so Metro doesn't fight itself on port 8081.

**Grab the numbers:** copy the three `RNWIND_BENCH_RESULT` lines out of the Metro log, or screenshot the on-screen summary.

## Tuning

Edit `examples/benchmark/packages/benchmark/src/config.ts`:

```ts
export const BENCHMARK_CONFIG = {
  SIZES: [10, 1000, 10_000] as const, // drop 10_000 if the simulator struggles
  RUNS_PER_SIZE: 5,                   // bump for tighter stddev
  DELAY_BETWEEN_RUNS: 100,            // ms between renders (platform settle)
} as const
```

## Known issues

- **nativewind v5 preview + bun workspace**: `react-native-css` (nativewind v5's transformer) needs `lightningcss@1.30.1`; rnwind pins `^1.32.0`. Bun hoists one version, so the nativewind5 app currently fails to bundle with `Error: failed to deserialize; expected ... Specifier, found ()`. An `overrides` block is declared in `apps/nativewind5/package.json` but bun doesn't cascade per-workspace overrides into transitive deps. Workaround TBD — track upstream at [expo/nativewind](https://github.com/expo/nativewind).
- **N=10_000 on older simulators**: rendering 10,000 views can take tens of seconds; if the simulator never emits the third `RNWIND_BENCH_RESULT` line, lower the top size in `config.ts`.
